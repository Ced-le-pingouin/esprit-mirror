Constructeur. 

La plupart des fonctions de cette classe sont des fonctions statiques, mais on peut tout de même instancier des objets de la classe, pour simuler différent types d'erreurs et les utiliser par exemple comme valeur de retour, pour ensuite les tester grâce à la fonction \doxyref{estErreur()}{p.}{class_erreur_2252580a9a3b9ce29b714c0b22c23257} (tout comme on peut en PHP 5 créer des objets de type {\tt Exception})

\begin{Desc}
\item[Paramètres:]
\begin{description}
\item[{\em v\_\-sTexte}]le texte qui décrit l'erreur \item[{\em v\_\-iNiveau}]le type d'erreur (voir constantes {\tt ERREUR\_\-}). Par défaut, il s'agit d'une erreur de type {\tt ERREUR\_\-AVERT} (contrairement à PHP, ou l'erreur par défaut pour {\tt trigger\_\-error()} est de type {\tt E\_\-USER\_\-NOTICE}, donc de moindre importance) \end{description}
\end{Desc}

\begin{Desc}

\begin{description}

\end{Desc}
\begin{Desc}
\item[Renvoie:]si {\tt v\_\-sTexte} est de type booléen, les erreurs déjà stockées s'il y en a, ou {\tt false} s'il n'y en a pas si {\tt v\_\-sTexte} est de type non-booléen, rien\end{Desc}
\begin{Desc}
\item[Note:]Cette fonction est censée être statique \end{Desc}
