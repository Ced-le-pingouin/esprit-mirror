\subsection{Description détaillée}
Interface dérivée de l'itérateur simple, qui permet en plus de savoir si un élément a des \char`\"{}enfants\char`\"{} (qui pourraient) donc eux aussi être \char`\"{}itérés\char`\"{}). 

Si un itérateur implémente cette interface, il pourra être passé par le décorateur \doxyref{IterateurRecursif}{p.}{class_iterateur_recursif}, qui effectue une itération en descendant également dans les enfants de l'itérateur de base \subsection*{Fonctions membres publiques}

\subsubsection{\setlength{\rightskip}{0pt plus 5cm}IterateurComposite::aEnfants ()}\label{class_iterateur_composite_1b161ee694a4b1acd98116fb012d971e}


Indique si l'élément courant est susceptible d'avoir des éléments \char`\"{}enfants\char`\"{}. 

Par exemple, dans le cas d'un système de fichiers, on peut considérer que cette méthode retournerait {\tt false} pour un fichier, mais {\tt true} pour un dossier (car il peut lui-même contenir des fichiers ou dossiers). Idem pour un tableau : si l'un de ses éléments est lui-même un tableau, la méthode \doxyref{aEnfants()}{p.}{class_iterateur_composite_1b161ee694a4b1acd98116fb012d971e} de cet élément retourne {\tt true}.
