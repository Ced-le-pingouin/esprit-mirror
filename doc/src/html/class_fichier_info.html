Référence de la classe FichierInfo

Graphe d'héritage de FichierInfo:

Liste de tous les membres

Description détaillée

Classe d'information sur les fichiers/dossiers du système de fichier: interrogations des caractéristiques, accès, etc.

Note:
Un objet de cette classe ne doit pas obligatoirement représenter un fichier existant, il s'agit seulement d'un chemin, qui peut ensuite être vérifié (il y a une exception : FichierInfo::retCheminReel())

Cette classe ne gère PAS les lectures/écritures DANS un fichier

Cette classe utilise des fonctions str...(), qui ne sont par défaut pas compatibles multibyte, mais je pense que pour les noms de fichiers/dossiers récupérés par l'OS, ça ne devrait pas poser de problèmes

À faire:
Il faudrait créer des méthodes pour:
  • transformer un chemin relatif en absolu (et vice versa) sans obligation que le chemin fourni existe réellement quand c'est possible;
  • déterminer la partie "racine" d'un chemin;
  • etc

FichierInfo::cheminContientDossier (  ) 

Indique si le chemin actuellement représenté par l'objet contient explicitement un dossier, ou s'il s'agit d'un simple nom de fichier.

Renvoie:
true si le chemin contient une référence à un dossier (par ex "/monDossier/monFichier.ext", ou même "./monFichier.ext" et "../monFichier.ext"), false sinon (par ex "monFichier.ext")
Note:
Je me sers de cette méthode pour différencier les chemins "./monFichier.ext" et "monFichier.ext", qui en temps normal sont "égaux" et représentent tous les deux "./monFichier.ext", mais qui dans certaines circonstances devraient pouvoir être différenciés pour modifier le comportement d'une méthode en fonction du type de chemin utilisé (par exemple dans renommer(), où le fichier sera toujours renommé dans son dossier d'origine si aucun dossier n'est spécifié explicitement dans le chemin de destination, mais où il sera *déplacé dans le dossier courant* si un "./" est spécifié devant le nom dans le chemin de destination)

Convertit, dans un chemin, les séparateurs de fichier/dossiers par défaut de l'OS courant, en séparateurs exigés par l'attribut sSeparateur de l'objet (détecté par defChemin() ou modifié par defSeparateur()).

Paramètres:

v_sSeparateurSrc le séparateur à remplacer. Par défaut, il s'agit du séparateur standard de l'OS
v_sSeparateurDst le séparateur qui doit remplacer celui spécifié par le paramètre v_sSeparateurSrc. Par défaut, il s'agit de l'attribut sSeparateur défini pour l'objet courant

Copie le fichier ou dossier représenté par l'objet courant dans un autre dossier.

Paramètres:

v_bEcraserExistant si true, les dossiers ou fichiers existants seront créés/copiés, même si la cible existait déjà, sans provoquer d'erreur ou d'avertissement. Si false, tout fichier cible existant au cours de la copie provoquera une erreur

v_bRecursif si true, les sous-dossiers du dossier source, et leur contenu, seront également copiés. Ce paramètre n'est donc utile qu'au cas ou le chemin représenté par l'objet courant est un dossier
v_fnCallback voir le paramètre du même nom de copierFichier()
Renvoie:
true si l'opération s'est bien déroulée, false dans le cas contraire

FichierInfo::separateurParDefaut ( $  v_sSeparateur = NULL  ) 

Définit un séparateur par défaut pour tous les objets de cette classe qui seront créés.

Si cette méthode n'est jamais appelée, le séparateur par défaut est celui de l'OS. Cette méthode retourne l'actuel séparateur par défaut si elle est appelée sans paramètre

Paramètres:
v_sSeparateur le séparateur par défaut à utiliser pour tous les objets de la classe. Si null (défaut), alors la méthode ne définit pas le séparateur par défaut, mais le retourne
Renvoie:
si v_sSeparateur est null, le séparateur par défaut actuel

Référencé par CTraverseurScorm::debutTraitement(), detecterSeparateur(), et CTraverseurScorm::finTraitement().

FichierInfo::supprimer ( $  v_bRecursif = FALSE  ) 

Supprime le fichier ou dossier représenté par l'objet courant.

Paramètres:
v_bRecursif si true, indique qu'il faut également effacer les fichiers et dossiers présents dans le dossier de base. Si false, le dossier sera supprimé uniquement s'il est vide. Ce paramètre n'est donc utile qu'au cas ou le chemin représenté par l'objet courant est un dossier
Renvoie:
true si l'opération s'est bien déroulée, false dans le cas contraire

Références estDossier(), existe(), Erreur::provoquer(), retChemin(), supprimerDossier(), et supprimerFichier().
