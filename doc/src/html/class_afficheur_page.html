Description détaillée

Contrôleur sommaire pour le système de templates.

Prend en charge les erreurs à afficher, l'utilisation d'un template par défaut, la mise à disposition des variables GET/POST/SESSION, et surtout les quelques méthodes de base et leur ordre d'appel (récupération et validation des données, exécution des actions demandées, affichage des erreurs et/ou des autres parties de la page)

AfficheurPage::afficherErreurs (  ) 

Affiche les messages d'erreur qui ont été déclarés pendant le traitement de la page.

Ils peuvent soit être présents au préalable dans le HTML, sous forme de blocs [erreur...] contenant le message voulu, soit sous la forme d'un seul bloc nommé [erreurs], dans lequel seront insérés les messages déclarés

Voir également:
declarerErreur()

Référencé par afficher().

AfficheurPage::declarerErreur ( $  sNomErreur,
$  bFatale = FALSE,
$  v_sTexte = '' 
)

Déclare qu'une erreur s'est produite.

Permet d'afficher des messages d'erreurs à la fin du traitement de la page, lors de l'affichage

Paramètres:
sNomErreur le nom de l'erreur déclarée. Si on omet v_sTexte, ce paramètre désigne le nom du bloc dans le template qui contient le message pour cette erreur, et qui sera donc affiché
bFatale indique si l'erreur est fatale, càd si elle empêchera l'exécution de l'action demandée pour la page et l'affichage des parties principales de la page
v_sTexte le message de l'erreur déclarée. S'il n'est pas spécifié, le nom de l'erreur est considéré comme le nom du bloc de template qui contient le message à afficher. S'il est spécifié, ce texte sera affiché dans le bloc [erreurs] du template en tant que §

Référencé par NavigateurFichiers::gererActions(), NavigateurFichiers::validerDonnees(), et CopierCollerFormation::validerDonnees().

Gère les actions demandées pour la page.
