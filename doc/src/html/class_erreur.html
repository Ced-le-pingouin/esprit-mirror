Liste de tous les membres

Description détaillée

Classe de gestion des erreurs.

Pour l'instant, cette classe est minimale et permet seulement de "provoquer" une erreur dans du code perso, étant donné qu'en PHP 4 les exceptions n'existent pas

Documentation des fonctions membres

Erreur::Erreur ( $  v_sTexte,
$  v_iNiveau = ERREUR_AVERT 
)

Constructeur.

La plupart des fonctions de cette classe sont des fonctions statiques, mais on peut tout de même instancier des objets de la classe, pour simuler différent types d'erreurs et les utiliser par exemple comme valeur de retour, pour ensuite les tester grâce à la fonction estErreur() (tout comme on peut en PHP 5 créer des objets de type Exception)

Paramètres:
v_sTexte le texte qui décrit l'erreur
v_iNiveau le type d'erreur (voir constantes ERREUR_). Par défaut, il s'agit d'une erreur de type ERREUR_AVERT (contrairement à PHP, ou l'erreur par défaut pour trigger_error() est de type E_USER_NOTICE, donc de moindre importance)

Erreur::erreurs ( $  v_sTexte = TRUE,
$  v_iNiveau = ERREUR_AVERT 
)

Stocke une nouvelle erreur ou retourne les erreurs déjà stockées.

Paramètres:
v_sTexte si de type booléen, la méthode retourne les erreurs déjà stockées. si de type non-booléen, le paramètre est considéré comme le texte d'une nouvelle erreur à stocker
v_iNiveau si le v_sTexte est de type non-booléen, ceci représente le niveau de l'erreur à stocker (voir constantes ERREUR_)
Renvoie:
si v_sTexte est de type booléen, les erreurs déjà stockées s'il y en a, ou false s'il n'y en a pas si v_sTexte est de type non-booléen, rien
Note:
Cette fonction est censée être statique

Erreur::mode ( $  v_iMode = NULL  ) 

Modifie ou retourne le mode de gestion des erreurs.

Paramètres:
v_iMode si null, le mode de de gestion actuel est retourné. Sinon, le paramètre est défini comme mode d'affichage (voir constantes ERREUR_MODE_...)
Renvoie:
