Template::afficher (  ) 

Affiche les données contenues actuellement dans le template.

Généralement appelé à la fin du script PHP qui utilise le template. Un remplacement automatique se fait avant l'affichage, sur les faux protocoles "racine://", "admin://", "commun://", "theme://", "javascript://", et "lib://", qui sont remplacés par, respectivement, les resultats des appels aux fonctions globales dir_root_plateform(), dir_admin(), dir_theme_commun(), dir_theme(), dir_javascript(), et dir_lib()

Template::defVariable ( $  looptag,
$  effacer = TRUE 
)

Retourne une "variable de template", càd un bloc entre balises [bloc+][bloc-].

Par exemple si le template contient un segment "[bloc+]contenu[bloc-]", un appel defVariable("bloc") renvoie le texte "contenu"

Paramètres:
looptag le nom du bloc dont les balises entourent la variable de template
Renvoie:
la variable trouvée. S'il n'existe pas de variable définie par des blocs [looptag+][lopptag-], renvoie null

Template::remplacer ( $  in,
$  out 
)

Remplace "simplement" des morceaux de template par d'autres.

Utilisé pour inclure des variables dans le template qui seront remplacées en PHP, par exemple un appel remplacer("{utilisateur}", $oProjet->oUtilisateur->retPseudo()) remplace toutes les chaînes "{utilisateur}" trouvées dans le template par le pseudo de l'utilisateur connecté

Paramètres:
in la chaîne de caractère à trouver dans le template
out la chaîne par laquelle on veut remplacer toutes les occurences de in
