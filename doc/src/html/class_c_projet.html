Référence de la classe CProjet

Liste de tous les membres

Description détaillée

Classe principale de la plate-forme.

Elle est utilisée dans la majorité des pages, et effectue diverses initialisations sur les utilisateurs, statuts, formations, etc

CProjet::CProjet ( $  v_bEffacerCookie = FALSE,
$  v_bRedirigerSiIncorrect = FALSE 
)

Constructeur.

Initialise l'objet principal du projet, généralement unique et global

Paramètres:
v_bEffacerCookie si true, le cookie actuel de l'utilisateur, contenant les infos de sa session, est effacé. Cela a pour effet, entre autres, de le déconnecter

CProjet::initFormationsUtilisateur ( $  v_bRechStricte = FALSE,
$  v_bStatutActuel = TRUE,
$  v_bDossierForms = FALSE 
)

Initialise un tableau (aoFormation) contenant les formations disponibles à l'utilisateur.

Paramètres:
v_bRechStricte si true, seules les formations pour lesquelles l'utilisateur a le statut exact demandé seront retournées si false, les formations pour lesquelles l'utilisateur a un statut inférieur à celui demandé seront aussi retournées
v_bStatutActuel si true, recherche les formations par rapport au statut actuel de l'utilisateur, càd la variable iStatutUtilisateur, qui peut avoir changé par rapport au statut enregistré (cookie) pour la session dans asInfosSession[SESSION_STATUT_UTILISATEUR], cette dernière étant utilisée si le paramètre est false.

À faire:
Pour le moment, le cryptage se fait par la fonction PASSWORD() de MySQL, ce qui était malheureusement une très mauvaise idée, car cette fonction ne devait être utilisée qu'en interne par MySQL, et a été modifiée dans MySQL 4.1, ce qui rendra les mots de passe déjà encodés incompatible avec la nouvelle version de la fonction
Idéalement, il faudrait donc crypter avec une fonction standard comme MD5() si on veut garder des mots de passe indécryptables ou des fonctions comme AES_CRYPT()/AES_DECRYPT() si on veut pouvoir récupérer le mot de passe à tout moment (ça éviterait le système tordu mis en place après coup, qui consiste à écrire une version décryptée du mot de passe de chaque utilisateur dans le fichier src/tmp/mdpncpte protégé contre la lecture, à chaque fois que quelqu'un se connecte et passe le login
